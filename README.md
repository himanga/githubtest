# foo-lib

A lightweight JavaScript utility for flexible value transformation.

## Documentation

The documentation site is published to GitHub Pages and has three tabs:

| Tab | Source | Description |
|---|---|---|
| **Home** | `docs/home.md` | Overview, quick-start, examples |
| **Help** | `docs/help.md` | Troubleshooting, FAQ, contributing |
| **API Reference** | `docs/naturaldocs/` | Auto-generated from NaturalDocs comments in `src/` |

Live site: `https://himanga.github.io/githubtest/`

## Local development

### 1 — Install NaturalDocs

Download [NaturalDocs 2.x](https://www.naturaldocs.org/download/) and unzip it to `NaturalDocs/` in the project root (the path expected by `npm run docs`).

### 2 — Generate API docs

```bash
npm run docs
```

NaturalDocs reads `src/foo.js` and writes HTML to `docs/naturaldocs/`.

### 3 — Serve locally

```bash
npm run docs:serve
```

Then open `http://localhost:3000`.

---

## Editing documentation

- **Home page** — edit `docs/home.md`
- **Help page** — edit `docs/help.md`
- **API reference** — edit the NaturalDocs comment blocks in `src/foo.js`, then re-run `npm run docs`

The tab shell (`docs/index.html`) fetches both markdown files at runtime, so there is no build step for content changes — just update the `.md` file and push.

---

## GitHub Pages setup

1. Go to **Settings → Pages** in your repository.
2. Set **Source** to `GitHub Actions`.
3. Push to `main` — the `deploy-docs.yml` workflow runs automatically.

The workflow:
1. Downloads NaturalDocs
2. Runs NaturalDocs to generate `docs/naturaldocs/`
3. Uploads the `docs/` folder as a Pages artifact
4. Deploys it to `https://<your-org>.github.io/foo-lib/`

---

## Adding new source files

1. Create your `.js` file in `src/` with NaturalDocs comment blocks.
2. NaturalDocs will pick it up automatically on the next build (no config change needed — `src/` is the input folder).
